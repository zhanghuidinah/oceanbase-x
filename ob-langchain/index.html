Context

Why AI Apps Need OceanBase

Data Silo Explosion

Vectors, JSON, full-text, GIS scattered across engines. Developers forced to stitch MySQL + ES + Pinecone into a Frankenstein architecture.

Freshness vs. Scale

Traditional vector DBs offer "eventual consistency" — stale RAG results. OceanBase guarantees strict ACID: queryable the millisecond data is written.

Glue Code Tax

Merging MySQL + ES + Vector DB results requires 500+ lines of Python. OceanBase does hybrid search in a single SQL query.

Operational Fatigue

Multiple clusters, multiple APIs, multiple failure modes — AI apps shouldn't be this complex.

Pain points: data silos, consistency, glue code, ops complexity. OceanBase solves all with one engine.

SDK

pyobvector — Two Python SDK Modes

Milvus-Compatible Mode

MilvusLikeClient class — API aligned with Milvus, zero-cost migration.

Python
from pyobvector import MilvusLikeClient

client = MilvusLikeClient(
    uri="127.0.0.1:2881",
    user="root@test"
)
# Create collection + vector index
client.create_collection(
    collection_name="docs",
    schema=schema,
    index_params=idx_params,
)
SQLAlchemy Hybrid Mode

ObVecClient class — vector ops + SQL queries in one, extends SQLAlchemy.

Python
from pyobvector import ObVecClient
from sqlalchemy import func

client = ObVecClient(
    uri="127.0.0.1:2881",
    user="root@test"
)
# ANN search + scalar filtering
res = client.ann_search(
    table_name="docs",
    vec_data=embedding,
    vec_column_name="embedding",
    distance_func=func.l2_distance,
    topk=5,
)
pyobvector offers two modes: Milvus-compatible (migration-friendly) and SQLAlchemy hybrid (flexible querying). Install: pip install pyobvector

RAG

End-to-End RAG Pipeline

Document Ingestion

LangChain DocumentLoader → TextSplitter → Embeddings → OceanBaseVectorStore

Hybrid Retrieval

User Query → Embedding → Vector + Full-text hybrid search → DBMS_HYBRID_SEARCH weighted fusion

Answer Generation

Retriever → Context assembly → LLM (GPT-4 / Qwen / DeepSeek) → Structured output

Evaluation & Iteration

LangSmith / RAGAS evaluation → Tune chunk size, retrieval weights, reranker strategy

Python — RAG Chain
from langchain_core.output_parsers import StrOutputParser
from langchain_core.runnables import RunnablePassthrough
from langchain_openai import ChatOpenAI

retriever = vs.as_retriever(search_kwargs={"k": 5})
llm = ChatOpenAI(model="gpt-4o")

rag_chain = (
    {"context": retriever | format_docs, "question": RunnablePassthrough()}
    | prompt
    | llm
    | StrOutputParser()
)

rag_chain.invoke("How does OceanBase achieve distributed consistency?")
Full RAG pipeline: load docs → chunk → embed → store → retrieve → generate. OceanBase replaces Pinecone/Weaviate as VectorStore.
Hybrid Search

DBMS_HYBRID_SEARCH Deep Dive

01
Scalar Filtering

WHERE clause narrows scope from millions to thousands

02
Coarse Ranking: Vector + Full-Text

HNSW vector ANN + FULLTEXT keyword match — low-cost parallel recall

03
Fine Ranking: Reranker

High-precision re-scoring with configurable vector/full-text weight ratio

SQL — Hybrid Search
CALL DBMS_HYBRID_SEARCH(
  table_name   => 'knowledge_base',
  query_text   => 'distributed transaction consistency protocol',
  vector_col   => 'embedding',
  fulltext_col => 'content',
  top_k        => 10,
  alpha        => 0.7   -- vector weight 0.7, full-text weight 0.3
);
Alpha controls vector vs full-text weighting. 0.7 = 70% semantic + 30% keyword. Tune per use case.

Quick Start

Up and Running in 5 Minutes

Shell — Docker Launch
# Start OceanBase CE (vector support requires V4.3.3+)
docker run -d --name ob \
  -p 2881:2881 \
  -e OB_MEMORY_LIMIT=6G \
  oceanbase/oceanbase-ce:4.3.5-lts
Shell — Install & Run
pip install langchain-oceanbase pyobvector langchain-openai

# Or use seekdb lightweight edition
docker run -d --name seekdb \
  -p 2881:2881 -p 2886:2886 \
  -v ./seekdb:/var/lib/oceanbase \
  oceanbase/seekdb:latest
Python — Minimal RAG Example
from langchain_oceanbase import OceanBaseVectorStore
from langchain_openai import OpenAIEmbeddings, ChatOpenAI

vs = OceanBaseVectorStore(
    embedding=OpenAIEmbeddings(),
    connection_args={"host":"127.0.0.1","port":"2881",
                     "user":"root@test","database":"test"},
)
vs.add_texts(["OceanBase is a distributed relational database"])
print(vs.similarity_search("What is OceanBase?", k=1))
Three steps to run: 1) Docker launch OB 2) pip install 3) Python code. seekdb is lighter, runs on 1C2G.
