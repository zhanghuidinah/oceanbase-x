痛点

传统 Agentic RAG 的多组件困境

传统方案依赖三个异构组件，系统越复杂，运维负担越重

关系型数据库

存储用户、应用配置、Agent 任务状态、知识库元数据——强事务性、结构化数据

向量数据库

存储 Context Chunks 经 Embedding 后的高维向量，支撑语义搜索

全文检索

构建倒排索引，支持关键词稀疏检索，保障精确匹配

运维噩梦

每套系统独立管理备份、升级、监控，任一环节故障可致全局性故障

数据同步困难

异构组件间数据一致性风险高，同步管道复杂易出错

成本攀升

系统越复杂，人力投入越大，风险越高，运维成本线性增长

传统 Agentic RAG 依赖关系型数据库 + 向量数据库 + 全文检索三个异构组件，运维复杂、数据同步困难。
v1.10.1

Dify v1.10.1 — 原生支持 OceanBase/seekdb

OceanBase 开源团队与顺丰 AI 技术平台组联合完成 Dify MySQL 兼容开发，Dify 开始尝试一体化数据库
MetaDB 层

Dify 已适配 MySQL 型 MetaDB，引入 DB_TYPE，一套迁移脚本兼容 PostgreSQL / MySQL / OceanBase，seekdb 可直接当 Dify 元数据库用

向量 & 检索层

OceanBase 已是 Dify 官方 VectorStore：支持向量检索、Hybrid Search（向量+全文）、metadata 过滤、score 阈值控制

运行环境 & 质量

Docker Compose 里有专门的 OB profile，起容器即可用；CI 里有真机 OB 实例跑向量相关测试保障

一体化精简

业务元数据、语义向量和全文索引统一存储检索，确保事务一致性，极大简化运维负担

seekdb 是 OceanBase 推出的 AI 原生轻量版，最低 1C2G 即可运行，支持向量与全文检索
Dify v1.10.1 正式兼容 MySQL/OceanBase/seekdb 作为元数据库，同时支持 OceanBase 作为向量存储，实现数据层彻底精简。

部署

Docker Compose 一键部署

01
克隆 Dify 仓库

git clone https://github.com/langgenius/dify.git && cd dify/docker

02
复制环境变量模板

cp .env.example .env

03
配置 .env（见右侧推荐配置）

设置 seekdb 同时作为元数据库和向量数据库

04
启动服务

docker compose up -d — 等待 seekdb Healthy 后自动迁移

.env — seekdb 作为元数据库 + 向量数据库（推荐）
# 元数据库配置
DB_TYPE=mysql
DB_USERNAME=root
DB_HOST=seekdb
DB_PORT=2881
DB_DATABASE=test

# 向量存储配置
VECTOR_STORE=oceanbase
OCEANBASE_VECTOR_HOST=seekdb
OCEANBASE_VECTOR_USER=root

# 启动 profile（仅 seekdb）
COMPOSE_PROFILES=seekdb
推荐方案：seekdb 同时作为元数据库和向量数据库，COMPOSE_PROFILES=seekdb 只启动必要容器。
灵活配置

三种配置方式，按需选择

情况 1：seekdb 仅作元数据库 
DB_TYPE=mysql
DB_HOST=seekdb
DB_PORT=2881
DB_DATABASE=test
# 向量存储保持默认
情况 2：seekdb 仅作向量数据库 
VECTOR_STORE=oceanbase
OCEANBASE_VECTOR_HOST=seekdb
OCEANBASE_VECTOR_USER=root
# 元数据库保持 PostgreSQL
情况 3：seekdb 同时作为元数据库和向量数据库（推荐） 
DB_TYPE=mysql
DB_HOST=seekdb
DB_PORT=2881
DB_DATABASE=test
VECTOR_STORE=oceanbase
OCEANBASE_VECTOR_HOST=seekdb
COMPOSE_PROFILES=seekdb

一个数据库搞定所有：元数据 + 向量 + 全文索引，运维零负担

三种配置方式灵活选择，推荐情况3一体化方案。COMPOSE_PROFILES=seekdb 只启动必要容器。
实战

通过 Dify 构建 AI 应用

01
设置模型供应商

设置 → 模型供应商 → 填入 API Key（如阿里云百炼）→ 选择系统推理模型和 Embedding 模型

02
创建知识库

知识库 → 创建 → 索引方式选"高质量" → 选择 Embedding 模型 → 上传文档 → 自动分块与嵌入

03
创建 ChatBot 应用

工作室 → 创建空白应用 → 聊天助手 → 添加知识库上下文 → 调试预览

04
发布应用

点击发布 → 运行 → 获得专属应用页面，可分享给团队使用

四步完成 AI 应用搭建：配模型 → 建知识库 → 建 ChatBot → 发布。
