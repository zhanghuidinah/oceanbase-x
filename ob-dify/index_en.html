Pain Points

Traditional Agentic RAG Multi-Component Challenges

Traditional solutions rely on three heterogeneous components — the more complex the system, the heavier the operational burden

Relational Database

Stores users, app configs, Agent task states, knowledge base metadata — strongly transactional, structured data

Vector Database

Stores high-dimensional vectors from embedded Context Chunks, powering semantic search

Full-Text Search

Builds inverted indexes for keyword sparse retrieval, ensuring precise matching

Operational Nightmare

Each system requires independent backup, upgrade, and monitoring — any failure can cause global outages

Data Sync Difficulties

High data consistency risks between heterogeneous components, complex sync pipelines prone to errors

Rising Costs

The more complex the system, the greater the human investment, higher risks, linear growth in operational costs

Traditional Agentic RAG relies on relational DB + vector DB + full-text search — three heterogeneous components with complex operations and data sync difficulties.
v1.10.1

Dify v1.10.1 — Native OceanBase/seekdb Support

OceanBase open source team and SF Express AI platform team jointly completed Dify MySQL compatibility development
MetaDB Layer

Dify now supports MySQL-type MetaDB with DB_TYPE. One migration script compatible with PostgreSQL / MySQL / OceanBase. seekdb can directly serve as Dify's metadata database.

Vector & Retrieval Layer

OceanBase is now an official Dify VectorStore: supports vector retrieval, Hybrid Search (vector + full-text), metadata filtering, and score threshold control.

Runtime & Quality

Dedicated OB profile in Docker Compose — containers ready to use; CI runs vector tests against real OB instances.

Unified Simplification

Business metadata, semantic vectors, and full-text indexes unified in one storage with transactional consistency, greatly simplifying operations.

seekdb is OceanBase's AI-native lightweight edition, runs on 1C2G minimum, supports vector and full-text search
Dify v1.10.1 officially supports MySQL/OceanBase/seekdb as MetaDB and OceanBase as vector store, achieving complete data layer simplification.

Deploy

Docker Compose One-Click Deployment

01
Clone Dify Repository

git clone https://github.com/langgenius/dify.git && cd dify/docker

02
Copy Environment Template

cp .env.example .env

03
Configure .env (Recommended Config)

Set seekdb as both MetaDB and Vector Store

04
Launch Services

docker compose up -d — wait for seekdb Healthy, auto-migration follows

.env — seekdb as MetaDB + Vector Store (Recommended)
# MetaDB Configuration
DB_TYPE=mysql
DB_USERNAME=root
DB_HOST=seekdb
DB_PORT=2881
DB_DATABASE=test

# Vector Store Configuration
VECTOR_STORE=oceanbase
OCEANBASE_VECTOR_HOST=seekdb
OCEANBASE_VECTOR_USER=root

# Launch profile (seekdb only)
COMPOSE_PROFILES=seekdb
Recommended: seekdb as both MetaDB and Vector Store. COMPOSE_PROFILES=seekdb starts only necessary containers.
Flexible Config

Three Configuration Options

Option 1: seekdb as MetaDB Only 
DB_TYPE=mysql
DB_HOST=seekdb
DB_PORT=2881
DB_DATABASE=test
# Vector store uses default
Option 2: seekdb as Vector Store Only 
VECTOR_STORE=oceanbase
OCEANBASE_VECTOR_HOST=seekdb
OCEANBASE_VECTOR_USER=root
# MetaDB keeps PostgreSQL
Option 3: seekdb as Both MetaDB and Vector Store (Recommended) 
DB_TYPE=mysql
DB_HOST=seekdb
DB_PORT=2881
DB_DATABASE=test
VECTOR_STORE=oceanbase
OCEANBASE_VECTOR_HOST=seekdb
COMPOSE_PROFILES=seekdb

One database for everything: metadata + vectors + full-text indexes, zero operational burden

Three flexible configuration options. Option 3 (unified) is recommended. COMPOSE_PROFILES=seekdb starts only necessary containers.
Practice

Build AI Apps with Dify

01
Configure Model Provider

Settings → Model Provider → Enter API Key (e.g., Alibaba Cloud Bailian) → Select system inference and embedding models

02
Create Knowledge Base

Knowledge → Create → Index Mode: "High Quality" → Select Embedding Model → Upload documents → Auto chunking and embedding

03
Create ChatBot App

Studio → Create Blank App → Chat Assistant → Add Knowledge Base context → Debug and preview

04
Publish App

Click Publish → Run → Get dedicated app page, shareable with your team

Four steps to build an AI app: Configure models → Create knowledge base → Build ChatBot → Publish.
