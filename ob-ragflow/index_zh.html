RAGFlow 简介

RAGFlow — 深度文档理解 RAG 引擎

深度文档解析

PDF / Excel / 图片 / PPT 多格式解析，保留结构化信息

可组分块

手动 / 自动 / Q&A / Table / Paper 多种分块策略

Agent 编排

可视化 Agent Flow，支持工具调用、条件分支、循环

Memory 管理

Agent 长期记忆，跨会话持久化，支持 CRUD 与检索

知识库治理

元数据批量管理、文件状态追踪、质量监控

开源 Apache 2.0

InfiniFlow 开源，社区活跃，支持私有化部署

RAGFlow 是 InfiniFlow 开源的深度文档理解 RAG 引擎，核心优势在于文档解析质量和可组分块。
PowerRAG

PowerRAG — OceanBase 版 RAGFlow

架构定位

基于 RAGFlow 基础框架，OceanBase 多模态一体化数据库作为底座，提供扩展能力与企业级增强。

OceanBase 底座

替代 MySQL 作为主数据库，统一存储元数据、向量、全文索引，一个引擎全覆盖。

多模态检索

SQL + NoSQL 一体化架构，充分利用 OceanBase 的高性能向量检索和全文搜索能力。

社区版开源

PowerRAG Community Edition 基于 RAGFlow，Apache 2.0 协议开源，Docker Hub 可直接拉取。

OceanBase 替换

OceanBase 替代 MySQL 的优势

维度 MySQL + 外挂向量库 OceanBase 统一引擎
元数据存储 MySQL OceanBase (MySQL 兼容)
向量检索 需外挂 Pinecone / Weaviate 内置 HNSW 向量索引
全文检索 需外挂 Elasticsearch 内置 FULLTEXT 索引
数据一致性 跨系统同步，有延迟 ACID 事务，零延迟
运维复杂度 3+ 系统运维 1 个引擎
迁移成本 — MySQL 协议兼容，零改造
OceanBase 替代 MySQL + 向量库 + ES 三件套，MySQL 协议兼容，迁移零改造。

快速开始

5 分钟部署 PowerRAG

Shell — Docker Compose 启动
# 克隆 PowerRAG 仓库
git clone https://github.com/oceanbase/powerrag.git
cd powerrag

# Docker Compose 启动（内置 OceanBase）
docker compose up -d

# 或直接拉取 Docker Hub 镜像
docker pull oceanbase/powerrag:latest
配置 LLM 模型
# 在 PowerRAG 控制台配置模型
# 设置 → 模型提供商 → 添加 Ollama / OpenAI / vLLM

# 推荐 Ollama 本地部署
ollama pull llama3.2    # Chat 模型
ollama pull bge-m3      # Embedding 模型

# PowerRAG 设置
Chat Model: llama3.2
Embedding Model: bge-m3
Base URL: http://host.docker.internal:11434
创建知识库 & 开始对话
1. 知识库 → 新建 → 上传文档（PDF/Markdown/Excel）
2. 选择分块策略 → 自动解析 & 嵌入
3. Chat → 关联知识库 → 开始 RAG 对话
4. Agent → 创建 Agent Flow → 添加 Memory 节点
三步上手：1) Docker 启动 2) 配置 LLM 模型 3) 创建知识库开始对话。支持 Ollama 本地模型。
